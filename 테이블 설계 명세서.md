# 📊 Chronicle Tactics - 테이블 설계 명세서

**기준:** 전략 턴제 수집형 모바일 RPG  
**유저 수:** 3만명 / **데이터 기간:** 6개월  
**갱신 유형:** Append (새 행 추가) / Update (기존 행 갱신) / Recalculate (재계산)

---

## 🗂 원천 테이블 (Raw Tables)
> GCS에 날짜별 파티션으로 저장. Airflow DAG 1이 매일 생성.

---

### 1. users
**설명:** 유저 기본 정보 테이블. 신규 가입 시 생성되며 일부 컬럼은 주기적으로 업데이트됨.  
**갱신 유형:** Append (신규 가입) + Update (last_login_date)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 고유 ID (PK) | 전체 |
| signup_date | DATE | 가입일 | DNU, 코호트 분석 |
| first_login_date | DATE | 첫 로그인 날짜 | 코호트 분석 |
| last_login_date | DATE | 마지막 로그인 날짜 | 이탈 예측 |
| country | STRING | 국가 (KR/US/JP/TW/TH) | 세그먼트 분석 |
| device_type | STRING | 디바이스 (ios/android) | 세그먼트 분석 |
| acquisition_channel | STRING | 유입 채널 (organic/paid/sns/store) | ROI, ROAS |
| acquisition_cost | FLOAT | 유저 획득 비용 (paid는 비용, organic은 0) | ROI, ROAS |
| ab_group | STRING | A/B 테스트 그룹 (A/B) | A/B 테스트 |

---

### 2. unit_stats
**설명:** 게임 내 유닛(기체) 메타 정보. 게임 업데이트 시 신규 유닛 추가됨.  
**갱신 유형:** Append (신규 유닛 출시)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| unit_id | INT | 유닛 고유 ID (PK) | - |
| unit_name | STRING | 유닛 이름 | - |
| unit_grade | STRING | 등급 (N/R/SR/SSR) | 가챠 분석 |
| unit_type | STRING | 타입 (공격/방어/지원/균형) | 조합 분석 |
| unit_series | STRING | 계열 (Series_A/B/C 등) | 시너지 분석 |
| strong_stage_type | STRING | 강한 스테이지 유형 | 스테이지 분석 |

---

### 3. pilot_stats
**설명:** 파일럿 메타 정보. 유닛에 탑승해 능력치를 부여하는 시스템.  
**갱신 유형:** Append (신규 파일럿 출시)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| pilot_id | INT | 파일럿 고유 ID (PK) | - |
| pilot_name | STRING | 파일럿 이름 | - |
| pilot_grade | STRING | 등급 (N/R/SR/SSR) | 가챠 분석 |
| pilot_skill | STRING | 특기 (공격강화/방어강화/회복 등) | 조합 분석 |

---

### 4. pilot_unit_synergy
**설명:** 파일럿-유닛 조합 시너지 정보. 특정 조합 시 보너스 발동.  
**갱신 유형:** Append (신규 시너지 추가)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| pilot_id | INT | 파일럿 ID (FK) | 조합 분석 |
| unit_id | INT | 유닛 ID (FK) | 조합 분석 |
| synergy_grade | STRING | 시너지 등급 (S/A/B/C) | 조합 분석 |
| bonus_rate | FLOAT | 보너스 배율 (예: 1.2 = 20% 강화) | 조합 분석 |

---

### 5. gacha_history
**설명:** 유저별 가챠 시도 상세 로그. 매일 새 행이 추가됨.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| gacha_date | DATE | 가챠 날짜 | 이벤트 반응 분석 |
| gacha_type | STRING | 가챠 유형 (normal/premium/event) | 매출 분석 |
| result_grade | STRING | 획득 등급 (N/R/SR/SSR) | 리텐션 분석 |
| result_unit_id | INT | 획득 유닛 ID (FK) | 픽업 분석 |
| cost_amount | INT | 소모 재화 | ARPU, ARPPU |

---

### 6. gacha_event_log
**설명:** 픽업 이벤트별 집계 데이터. 신규 유닛 출시 시 가챠 반응 측정.  
**갱신 유형:** Append (이벤트 종료 후) + Update (진행 중 집계 갱신)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| event_id | INT | 이벤트 고유 ID (PK) | - |
| unit_id | INT | 픽업 유닛 ID (FK) | 픽업 분석 |
| event_start_date | DATE | 이벤트 시작일 | 이벤트 분석 |
| event_end_date | DATE | 이벤트 종료일 | 이벤트 분석 |
| total_gacha_count | INT | 이벤트 기간 총 가챠 수 | 매출 분석 |
| unique_user_count | INT | 참여 유저 수 | DAU 연계 분석 |

---

### 7. stage_clear_log
**설명:** 스테이지 클리어 시도 로그. 유저의 게임 진행 상황 추적.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| stage_id | INT | 스테이지 번호 (1~200) | 이탈 구간 분석 |
| attempt_count | INT | 시도 횟수 | 난이도 분석 |
| clear_yn | BOOL | 클리어 여부 | 리텐션 분석 |
| clear_time_sec | INT | 클리어 소요 시간(초) | 난이도 분석 |
| used_item_yn | BOOL | 아이템 사용 여부 | 매출 분석 |
| event_date | DATE | 시도 날짜 | 코호트 분석 |

---

### 8. stage_unit_log
**설명:** 스테이지별 유닛+파일럿 사용 로그. 어떤 조합이 어떤 스테이지에 쓰이는지 추적.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| stage_id | INT | 스테이지 번호 | 유닛 활용 분석 |
| unit_id | INT | 사용 유닛 ID (FK) | 유닛 메타 분석 |
| pilot_id | INT | 탑승 파일럿 ID (FK) | 조합 분석 |
| slot_position | INT | 파티 슬롯 위치 (1~5) | 편성 분석 |
| clear_yn | BOOL | 클리어 여부 | 유닛 성능 분석 |
| event_date | DATE | 전투 날짜 | - |

---

### 9. party_composition_log
**설명:** 파티 편성 조합 로그. 유닛 5개 조합과 시너지 유형 기록.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| stage_id | INT | 스테이지 번호 | 조합 분석 |
| unit_ids | STRING | 편성 유닛 ID 목록 (예: "1,5,12,7,3") | 조합 분석 |
| synergy_type | STRING | 조합 시너지 유형 (공격형/균형형/방어형) | 클리어율 분석 |
| clear_yn | BOOL | 클리어 여부 | 조합 성능 분석 |
| event_date | DATE | 편성 날짜 | - |

---

### 10. session_log
**설명:** 유저 세션 로그. 로그인/로그아웃 시각 기록. ACU/CCU/PCU 계산에 사용.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| session_id | STRING | 세션 고유 ID (PK) | - |
| user_id | STRING | 유저 ID (FK) | - |
| login_time | TIMESTAMP | 접속 시작 시각 | CCU, PCU |
| logout_time | TIMESTAMP | 접속 종료 시각 | CCU, PCU |
| session_duration | INT | 세션 지속 시간(분) | ACU, 플레이타임 분석 |

---

### 11. user_unit_inventory
**설명:** 유저별 보유 유닛 현황. 강화할 때마다 enhance_level이 갱신됨.  
**갱신 유형:** Append (신규 획득) + Update (강화 시 enhance_level 갱신)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| unit_id | INT | 유닛 ID (FK) | 보유 현황 분석 |
| acquired_date | DATE | 획득일 | 가챠 분석 |
| enhance_level | INT | 강화 레벨 (0~10) | 과금 분석 |
| is_favorite | BOOL | 즐겨찾기 여부 | 선호 유닛 분석 |

---

### 12. user_daily_snapshot
**설명:** 유저 일별 상태 스냅샷. 매일 전체 유저 상태를 기록해 시계열 분석 가능.  
**갱신 유형:** Recalculate (매일 덮어쓰기)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| snapshot_date | DATE | 스냅샷 날짜 | 시계열 분석 |
| level | INT | 현재 레벨 | 이탈 예측 |
| total_playtime | INT | 누적 플레이타임(분) | LTV 예측 |
| total_gacha | INT | 누적 가챠 횟수 | ARPPU |
| total_spent | FLOAT | 누적 결제금액 | LTV, ARPPU |
| last_login_date | DATE | 마지막 로그인 | 이탈 예측 |
| churn_risk_score | FLOAT | 이탈 위험 스코어 (모델 예측값) | 이탈 예측 |

---

### 13. daily_game_metrics
**설명:** 게임 전체 지표 일별 집계. 매일 자정 Airflow가 전날 지표를 계산해서 저장.  
**갱신 유형:** Recalculate (매일 새 행 추가)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| metric_date | DATE | 집계 날짜 | - |
| DAU | INT | 일일 활성 유저 수 | DAU 트렌드 |
| DNU | INT | 일일 신규 유저 수 | DNU 트렌드 |
| MAU | INT | 월간 활성 유저 수 | MAU 트렌드 |
| PCU | INT | 최대 동시 접속자 수 | PCU 트렌드 |
| ACU | FLOAT | 평균 동시 접속자 수 | ACU 트렌드 |
| total_revenue | FLOAT | 일 총 매출 | ARPU, ARPPU |
| total_gacha_count | INT | 일 총 가챠 횟수 | 가챠 분석 |
| PUR | FLOAT | 결제 유저 비율 | PUR 트렌드 |
| ARPU | FLOAT | 전체 유저 1인당 평균 매출 | ARPU 트렌드 |
| ARPPU | FLOAT | 결제 유저 1인당 평균 매출 | ARPPU 트렌드 |

## 🔬 분석용 테이블 (Analytics Tables)

### 14. events (분석용)
**설명:** 원천 로그들을 유저×날짜 단위로 집계한 분석용 테이블.  
**갱신 유형:** Recalculate (주간 갱신)  
**원천:** session_log + stage_clear_log + gacha_history

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| event_date | DATE | 날짜 | - |
| login_yn | BOOL | 로그인 여부 | DAU |
| playtime_minutes | INT | 당일 총 플레이타임 | 이탈 예측 |
| level | INT | 현재 레벨 | 이탈 예측 |
| stage_clear_yn | BOOL | 당일 스테이지 클리어 여부 | 리텐션 분석 |
| gacha_result | STRING | 당일 최고 가챠 결과 등급 | 리텐션 분석 |
| gacha_try_count | INT | 당일 가챠 시도 횟수 | ARPU |
| item_use_count | INT | 당일 아이템 사용 횟수 | 매출 분석 |
| session_count | INT | 당일 세션 수 | 이탈 예측 |
| consecutive_login_days | INT | 연속 로그인 일수 | 이탈 예측 |

**갱신 유형:** Append + Recalculate (누적값 갱신)

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |

| purchase_amount | FLOAT | 결제 금액 | ARPU, ARPPU |
| item_category | STRING | 아이템 카테고리 (gacha/costume/stamina/package) | 매출 분석 |
| is_first_purchase | BOOL | 첫 결제 여부 | LTV 예측 |
| cumulative_amount | FLOAT | 누적 결제 금액 | LTV 예측 |
| payment_count | INT | 총 결제 횟수 | ARPPU |

### 16. marketing_events (분석용)
**설명:** 마케팅 캠페인 노출/클릭/전환 로그. A/B 테스트 핵심 테이블.  
**갱신 유형:** Append

| 컬럼 | 타입 | 설명 | 활용 지표 |
|------|------|------|-----------|
| user_id | STRING | 유저 ID (FK) | - |
| campaign_date | DATE | 캠페인 날짜 | - |
| impression_yn | BOOL | 노출 여부 | CTR |
| click_yn | BOOL | 클릭 여부 | CTR |
| conversion_yn | BOOL | 전환(결제) 여부 | CVR |
| reward_amount | FLOAT | 지급 보상 금액 (A: 1000, B: 1500) | A/B 테스트 |
| ad_cost | FLOAT | 광고 노출당 비용 | CPM, ROAS |
| ad_revenue | FLOAT | 광고 발생 수익 | eCPM, ROAS |

## 📐 지표 계산 공식 요약

| 지표 | 계산 공식 | 사용 테이블 |
|------|-----------|-------------|
| DAU | 특정일 login_yn=True 유저 수 | events |
| MAU | 특정월 로그인 unique 유저 수 | events |
| DNU | 특정일 signup_date 유저 수 | users |
| PCU | 시간대별 동시접속 최댓값 | session_log |
| ACU | 하루 평균 동시접속자 | session_log |
| CCU | 특정 시각 동시접속자 | session_log |
| PUR | 결제 유저 수 / DAU | purchases + events |
| ARPU | 일 매출 / DAU | purchases + events |
| ARPPU | 일 매출 / 결제 유저 수 | purchases |
| CPM | (ad_cost / impression 수) × 1000 | marketing_events |
| eCPM | (ad_revenue / impression 수) × 1000 | marketing_events |
| ROI | (매출 - acquisition_cost) / acquisition_cost × 100 | users + purchases |
| ROAS | 매출 / ad_cost × 100 | purchases + marketing_events |
| LTV | 모델 예측값 | user_daily_snapshot |
